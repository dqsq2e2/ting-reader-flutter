# Changelog

## 1.1.4 - 2026-07-06

- 书籍详情页操作栏支持显示插件扩展入口，有扩展时以"更多"菜单按钮形式内嵌在操作栏中，与其他操作按钮等宽排列。
- 插件扩展支持自定义图标，书籍详情、书架等入口的插件按钮图标更直观。
- 登录页支持删除已保存的服务器配置，换服务器时不再需要手动清理。
- 书籍详情页操作按钮改为按可用宽度自适应等分排列，小屏下不再溢出。

## 1.1.2 - 2026-07-02

- 修复 Windows / Linux 桌面端插件 Web UI 显示“WebView 不可用”的问题，客户端现在接入桌面 WebView 实现，可打开 `web_container` 插件界面。
- 修复 Windows WebView2 忽略 `loadHtmlString(baseUrl)` 导致插件资产被 CSP 拒绝、页面显示“拒绝连接”的根因；Windows 端改为直接加载插件资产并注入插件桥接脚本。
- 插件 Web UI 初始化上下文新增主题信息，Windows 顶层 WebView 会按应用当前亮色 / 深色设置同步 `color-scheme`，避免插件页面跟随系统主题而与应用设置不一致。
- 插件管理页将 `tool_provider` 标签从“AI 工具”改为“工具能力”，避免 FFmpeg Provider 等基础工具插件被误标为 AI。
- Release 构建流程补齐 Linux 桌面 WebView 所需的 WebKitGTK 构建依赖。

## 1.1.1 - 2026-07-01

- 客户端全面接入多语言：新增中文 / 英文界面切换，登录、我的、下载、通知、书籍详情、播放器等页面文案全部走本地化，插件描述、配置项和事件通知也会按当前语言展示。
- 界面语言设置会同步到服务端账号，同时本地保留，换设备登录也能沿用同一语言。
- 同步服务端 1.4.8 新能力：支持 RSS/播客存储库、官方插件商店和 AI 元数据刮削插件的展示与管理入口。
- 插件详情页展示官方签名 / 未验证来源标记，安装未验证的 `.tr` 插件时会弹出显式确认，避免误装第三方包。
- 插件卡片支持通过官方商店一键升级，并展示最新版本、当前版本、发布仓库和最低客户端版本要求。
- 敏感的插件配置项（如 AI 密钥、Webhook Token）在编辑时以掩码显示，留空保存即保留原值，不再在弹窗里回显明文。
- 内置的“文档阅读”入口支持通过插件扩展加载自定义视图，配合服务端插件扩展能力提供更多阅读场景。
- 优化启动流程提示语与多语言下的滚动标题、按钮换行表现，长文案在英文界面下不再被截断或撑破布局。


## 1.1.0 - 2026-06-25

- 重做章节管理弹窗，正文/番外切换贴近标题，列表默认轻量展示，点编辑再进入单章编辑，避免移动端大列表误触和信息过载。
- 优化章节分组标签：100 章以内不显示分组行，没有番外时不显示正文/番外切换，分组行不再重复提供跳转入口。
- 统一分组横向滚动体验，章节列表、下载章节列表、播放器章节弹窗和搜索筛选在桌面端鼠标悬浮时显示左右切换按钮，触摸滑动不触发。
- 修复搜索筛选词条在不同分类行里被居中导致起点不一致的问题，所有筛选项现在统一从同一列开始。
- 美化批量选择控件，统一全选、章节勾选框和已选数量在大小屏下的尺寸与排布，避免小屏控件独占整行。
- 调整书籍详情标签折叠与章节列表字重，小屏字体和间距随宽度收敛，整体视觉更接近 Web 端。
- Android APK 仅构建 `armeabi-v7a` 与 `arm64-v8a`，去除用不上的 x86_64 架构以减小体积；AAB 保留默认全架构用于商店分发。

## 1.0.9 - 2026-06-24

- 修复插件安装、用户权限保存和书架筛选在部分场景下请求参数不生效的问题。
- 优化沉浸式播放页竖屏布局，封面尺寸和底部留白会随屏幕高度、安全区动态调整，并保留滚动兜底。

## 1.0.8 - 2026-06-23

- 优化导航归属：下载页归入“我的”高亮；搜索会按来源从首页或书架返回；移除多余的顶层设置导航状态。
- 文案调整：收听历史统一改为“我的历史”。

## 1.0.7 - 2026-06-23

- 重构覆盖率：`dart analyze --fatal-warnings` 通过，public API 100% 兼容（仅新增 `DialogLabel` 一个共享组件），功能 0 改动。

## 1.0.6 - 2026-06-23

- 修复 Android 永久音频焦点丢失后无法恢复的根因：`audio_session` 收到
  `AUDIOFOCUS_LOSS` 会主动 abandon，并且不会再发送 interruption end；客户端
  现在会等待外部媒体真正结束后重新申请焦点并恢复播放。
- 移除每次播放前 `setActive(false)` 再激活的错误重置逻辑，避免主动放弃焦点
  与异步中断回调竞争，导致手动继续播放后再次被打断时退化为混音。
- 通知栏、锁屏和耳机按键等所有后台播放入口统一先申请音频焦点，避免绕过
  `PlayerState` 后在无焦点状态下继续播放。
- 耳机断连除 `AUDIO_BECOMING_NOISY` 外同时监听输出设备移除，覆盖有线、
  蓝牙、USB 和助听设备，并确保断连后只暂停、不自动恢复。
- “被其他应用暂停后自动恢复”保持默认始终开启，并清理旧版本遗留的本地设置。

## 1.0.5 - 2026-06-23

- 修复被通话、其他应用等短暂打断后播放无法自动恢复的问题，改由 just_audio 内置的中断管线统一处理 begin/end 事件。
- 修复耳机断开连接时播放不会自动暂停的问题，让 audio_session 的 noisy 接收器在每次申请音频焦点时正确注册。
- 将"被其他应用暂停后自动恢复"从设置中移除，默认始终启用，无需用户切换。
- 清理项目中残留的 lint 警告（`withOpacity`、`Color.red/green/blue`、`PopScope.onPopInvoked`、`ColorScheme.background`、`DropdownButtonFormField.value` 等弃用 API）。
