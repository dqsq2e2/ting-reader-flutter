# Changelog

## 1.0.4 - 2026-06-22

- 修复插件卡片把 `作者/仓库` 短路径直接当作网址打开的问题，统一跳转对应的 GitHub 仓库。
- 插件卡片仓库入口改用 GitHub 图标，与 Web 端保持一致。
- 修复恢复播放时播放日志可能从 0 秒开始记录的问题，按真实恢复位置上报播放起点。

## 1.0.3 - 2026-06-22

- 统一在线书籍、播放器队列与本地下载的章节播放顺序，正文始终排列在番外之前。
- 修复正文与番外使用相同章节序号时，切换章节、连续播放或从下载页播放可能进入错误条目的问题。
- 为章节排序增加稳定的同序号处理，避免不同入口生成的播放队列顺序不一致。

## 1.0.2 - 2026-06-22

- 同步服务端最新 Webhook 配置字段，支持自定义请求头和 Body 模板，并在编辑、启停时完整保留配置。
- 同步 Web 端 Webhook 添加与编辑流程，增加常见模板、测试发送、请求结果和实际请求体查看。
- 优化 Webhook 设置弹窗的深色模式配色，提升输入框、事件选项、说明文字和模板编辑区的可读性。
- 修复 Webhook 启用开关关闭后状态文案仍显示“开启”的问题。

## 1.0.1 - 2026-06-20

- 修复 Windows 桌面端中文字体渲染发虚、同词字重不一致的问题，随 Windows 发行包加载静态 Noto Sans CJK SC 字体。
- 统一客户端普通文本与控件文案的字重策略，减少正文、按钮、副标题等位置的视觉粗细跳变。
- 修复深色模式下“我的”页面统计数字颜色仍按浅色主题显示的问题。
- “我的下载”页面增加返回入口，保持与其他二级页面一致。
- iOS 发布产物改为标准 IPA 结构，可通过 AltStore、SideStore 等工具自行签名安装。
- 修复 iOS 锁屏播放器首次播放时未显示章节标题、作者和书籍封面的问题。

## 2026-06-18

- 初始化独立 Flutter 客户端仓库资料。
- 补充全平台构建工作流和固定 Release 产物命名。
- 增加 Linux `just_audio_media_kit` 播放后端。
- 补齐 macOS Release 网络客户端 entitlement。
- 记录各平台播放器、后台媒体控制和下载能力边界。
- 关于界面支持检查客户端最新版本并打开下载/安装流程。
- 官网链接和插件仓库链接支持跳转系统浏览器。
